# 에브리타임 시간표 파싱 가이드

OCR 방식 대신 **에브리타임 URL을 직접 파싱**하여 시간표를 추출하고 분석하는 방법입니다.

## 🎯 주요 기능

### 1. **단일 URL 검증 및 분석** (`check_everytime_url.py`)
- 에브리타임 공용 시간표 URL 검증
- HTML 페칭 (Playwright 또는 정적 HTML)
- 시간표 자동 파싱
- 시간표 데이터 검증
- 통계 정보 표시

### 2. **다중 URL 비교 분석** (`analyze_everytime_urls.py`)
- 여러 학생의 시간표 동시 파싱
- 개별 시간표 출력
- **공통 자유 시간 자동 계산**
- JSON 형식으로 결과 저장

  "https://everytime.kr/@XXXXX" \
  "https://everytime.kr/@YYYYY"

# 사용자 정의 이름으로 3명 분석
python analyze_everytime_urls.py \
  "https://everytime.kr/@XXXXX" \
  "https://everytime.kr/@YYYYY" \
  "https://everytime.kr/@ZZZZZ" \
  --names "김철수" "이영희" "박민준"

## 🔍 코드 구조

### `check_everytime_url.py` - 단일 URL 검증
```python
# 핵심 함수
check_everytime_url(url: str, timeout: int = 10) -> bool
    ↓
    1. URL 검증 (형식 확인)
    2. HTML 페칭 (Playwright/정적)
    3. 시간표 파싱
    4. 시간표 표시
    5. 데이터 검증
    6. 통계 계산
```

### `analyze_everytime_urls.py` - 다중 URL 분석
```python
# 핵심 함수
check_multiple_urls(
    urls: List[Tuple[str, str]],
    timeout: int = 10,
    min_free_duration: int = 30,
    output_file: str = None
) -> bool
    ↓
    1. 모든 URL 파싱
    2. 개별 시간표 출력
    3. 통계 계산
    4. 공통 자유 시간 계산
    5. 결과 JSON 저장 (옵션)
```

## 🛠️ 내부 구현

### 사용되는 핵심 클래스
- **`EverytimeTimetableParser`** (`src/services/everytime_parser.py`)
  - `validate_everytime_url()`: URL 형식 검증
  - `fetch_html()`: HTML 페칭 (Playwright/정적 방식 자동 선택)
  - `parse_html_to_pairs()`: HTML → 시간대 튜플 변환

### 데이터 형식
```python
# 시간대 튜플: (요일_인덱스, 시작분, 종료분)
intervals: List[Tuple[int, int, int]]
# 예: (0, 540, 630)  # 월요일 09:00-10:30

# 요일 인덱스
DAYS = ['월', '화', '수', '목', '금', '토', '일']  # 0-6
```

## ✨ 주요 특징

### 자동 자유 시간 계산
```
학생1: 월 09:00-10:30 수업
학생2: 월 10:00-11:30 수업

공통 자유 시간:
  • 08:00-09:00 ✅ (둘 다 자유)
  • 10:00-10:30❌ (학생1이 수업)
  • 10:30-11:00 ✅ (둘 다 자유)
  • 11:00-11:30 ❌ (학생2가 수업)
  • 11:30-22:00 ✅ (둘 다 자유)
```

### 검증 항목
- ✅ 시간 범위 (08:00-23:59)
- ✅ 요일 범위 (월-일)
- ✅ 수업 길이 (4시간 이상 감지)
- ✅ 시간 겹침 (같은 요일 중복 체크)
- ✅ 최소 수업 길이 (15분 이상)

## 🚀 활용 사례

### 1. 스터디 그룹 일정 조율
```bash
python analyze_everytime_urls.py \
  "https://everytime.kr/@김철수" \
  "https://everytime.kr/@이영희" \
  "https://everytime.kr/@박민준" \
  --names 철수 영희 민준 \
  --output study_group.json
```
→ 3명이 함께 만날 수 있는 시간을 자동으로 찾음

### 2. 프로젝트 팀 구성
같은 시간표 패턴을 가진 학생들을 자동으로 매칭

### 3. 학사 일정 자동화
API에 URL을 제출하면 자동으로 시간표 추출 및 자유 시간 계산

## ⚙️ 설정

### 타임아웃 조정
- 기본값: 10초
- Playwright 렌더링이 필요한 경우: 15-20초 권장

### 최소 자유 시간
- 기본값: 30분
- `analyze_everytime_urls.py`에서 `min_free_duration` 파라미터 조정 가능

## 🐛 트러블슈팅

### "Invalid Everytime timetable URL" 에러
```
✅ 올바른 형식: https://everytime.kr/@문자숫자조합
❌ 잘못된 형식: https://everytime.kr/timetable?id=xxx
```

### "Timetable columns not found in HTML" 에러
- 시간표가 비어있을 수 있음
- HTML 구조가 변경되었을 가능성
- Playwright 렌더링 시간 초과

### Playwright 설치 필요
```bash
pip install playwright
playwright install chromium
```

## 📈 성능

- **단일 URL**: ~10초 (Playwright 렌더링 포함)
- **다중 URL (n명)**: ~10n초 (병렬 처리로 최적화 가능)
- 메모리: ~50MB (URL당)
